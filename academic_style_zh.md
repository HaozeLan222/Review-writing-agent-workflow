# 中文学术表达润色规则

本文件用于在综述论文写作后期统一中文表达风格。它只处理语言清晰度、术语一致性和段落衔接，不新增事实、数据、案例或引用。

## 适用场景

- 中文综述论文段落润色
- 摘要、引言、相关工作和结论的表达统一
- 将过于机械或堆叠的句子改得更自然
- 检查术语、引用锚点和段落逻辑是否一致

## 基本原则

- 保留原文的事实、论点、引用和结构。
- 不新增文献、实验数据、应用案例或作者观点。
- 不删除必要的引用锚点。
- 不把不确定判断改写成确定结论。
- 优先保证学术准确性，其次才是语言流畅度。

## 润色流程

### 1. 术语统一

首次出现的重要概念建议采用“中文术语（英文全称, 缩写）”格式，后续可以使用中文术语或缩写。

示例：

```text
深度强化学习（Deep Reinforcement Learning, DRL）
```

同一篇文章中，同一概念不要反复切换译名。

### 2. 句子清理

优先处理以下问题：

- 句子过长，多个逻辑关系堆在一起。
- 同一段反复使用“显著”“关键”“重要”等泛化词。
- 转折、因果和递进关系不清楚。
- 中英文术语混排不规范。
- 段落开头和结尾过于模板化。

### 3. 引用保护

润色时必须保留引用锚点，例如 `[P01]`。如果拆分句子，应把引用移动到仍然承载事实陈述的位置。

示例：

```text
原句：该方法在多个任务上取得更好结果，并说明反馈机制能够提升搜索效率 [P03]。
改写：该方法在多个任务上取得了更好的结果 [P03]。作者据此认为，反馈机制可以提升搜索过程的效率 [P03]。
```

### 4. 段落逻辑

每个段落尽量只承担一个功能：

- 背景段：说明问题为什么存在。
- 方法段：说明某类方法如何工作。
- 对比段：说明不同方法的差异和适用边界。
- 评价段：说明已有研究的贡献、局限和待解决问题。

## 输出要求

当用户要求润色时，只输出修改后的正文。除非用户明确要求解释，否则不要附加改写说明、候选版本或额外建议。
